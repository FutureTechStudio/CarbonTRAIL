# Product Requirements — CarbonTrail AI

## Objective
CarbonTrail AI helps people understand, track, and reduce the carbon footprint of day-to-day activities through a smart assistant and visual carbon journey.

## Target user
A busy individual who wants awareness but does not want long forms. The user may commute, eat home/restaurant/delivery food, use AC/electricity, shop online, and generate waste.

## Core problem
Daily carbon impact is invisible. Most people do not know how commute, meals, electricity, deliveries, shopping, and waste add up. Standard calculators are too static and form-heavy.

## Product solution
Sprout AI learns the user's routine via lightweight check-ins. The user can answer with chips or natural language. The app translates activities into estimated CO2, confidence, LeafPoints, Green Impact, and visual trail effects.

## Core user loop
1. Start as Guest.
2. Complete quick baseline.
3. Sprout AI asks what changed today.
4. User replies by chips or typing.
5. App creates activity entries.
6. Carbon engine estimates footprint.
7. Today Trail updates visually.
8. LeafPoints reward detail sharing.
9. Green Impact shows possible savings.
10. Weekly Forest shows progress.

## MVP success criteria
- User can start without signup.
- Baseline profile is created.
- Saved profile is reused to avoid repeated questions.
- User can type a day update.
- Parser maps typed text to activity categories.
- Today Trail shows confirmed/assumed/estimated activities.
- Skipped day can be auto-filled from routine.
- Profile is editable.
- LeafPoints and Green Impact are separate.
- App is responsive and builds successfully.

## Out of scope for MVP
- Real auth/backend.
- Bank/spend integration.
- GPS/device tracking.
- Push notifications.
- Social leaderboard.
- Certified carbon accounting.
- Heavy AI dependency.
- Heavy assets or videos.
